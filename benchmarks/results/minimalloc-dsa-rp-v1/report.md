# DSA benchmark results

Generated by `dsa-suite` (run label: minimalloc-dsa-rp-v1). Raw per-run data is in `results.jsonl`; aggregated long-form data is in `summary.csv`, and per-instance constraint counts are in `features.csv`.

Configuration: seeds `0`, search budget `500`, local-search restarts `1`, MiniMalloc timeout `60000 ms` per instance. Runtime values are machine-dependent medians; placement metrics and validity are independently recomputed. `placement_valid` checks address geometry while ignoring capacity only for best-effort diagnostics; `solution_valid` includes the original capacity constraints.

Regenerate from the repository root:

```bash
./build/dsa-suite \
  --dsa-rp 'benchmarks/minimalloc-dsa-rp' \
  --output-dir 'benchmarks/results/minimalloc-dsa-rp-v1' \
  --run-label 'minimalloc-dsa-rp-v1' \
  --seeds '0' \
  --methods 'canonical_branch_and_bound,canonical_greedy,capacity_two_exact,cypress_relaxation,first_fit,implicit_hitting_set,local_search,promote_repair,reuse_penalty_local_search,reuse_penalty_portfolio,scale_separated_grid_dp,span_one_min_cost_flow,treewidth_partition_dp,tvm_hill_climb,unit_low_rank_rounding,unit_random_coloring' \
  --iterations '500' \
  --restarts '1' \
  --stagnation '100' \
  --deterministic-repetitions '5' \
  --minimalloc-timeout-ms '60000' \
  --no-minimalloc \
  --no-core-relaxations
```

## Generic DSA-RP

These are capacity-constrained DSA instances with weighted soft anti-alias edges; they do not use PyPTO pipeline relaxation. Each cell reports `reuse_cost @ peak` for the best validated run. Timeout incumbents are feasible placements, not optimality certificates.

All inputs in this snapshot use a deterministic synthetic access schedule. They stress the DSA-RP algorithms; they are not evidence for hardware synchronization cost.
