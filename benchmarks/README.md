# Benchmark data

`corpus.csv` is the checked-in inventory of all nontrivial, unique PyPTO and
PyPTO-Lib DSA instances. It is generated by `dsa-suite` as `features.csv`; one
row describes one structured JSON input.

The main columns cover provenance, buffer count, minimum/maximum/total buffer
bytes, number of distinct sizes, whether every buffer has the same size,
alignment, temporal conflict and reuse-candidate counts,
memory-space capacities, raw interval-live bytes, and PyPTO alias/pipeline
structure. `uniform_buffer_size=true` is the unweighted/unit-weighted special
case of DSA.

The corpus excludes captures with no placement choice: instances whose buffers
can all share one address and that have no explicit separation, pinning,
colocation, temporal-exclusion, or reuse-cost structure. Zero temporal
conflicts alone is not an exclusion criterion because PyPTO-specific
constraints can still make such an instance meaningful.

PyPTO pool names map to hardware memory spaces as follows:

| Pool | Hardware space |
| --- | --- |
| `Vec` | UB |
| `Mat` | L1 |
| `Left` | L0A |
| `Right` | L0B |
| `Acc` | L0C |

A document may contain several independent pools. Therefore
`pool_capacities_bytes` and `max_interval_live_bytes_by_space` retain one value
per space instead of collapsing L1 and the three L0 memories into one number.
`max_live_capacity_ratio` is the largest raw interval-live/capacity ratio among
the document's pools; it is a pressure indicator, not a solved placement peak.

`capture/` contains coverage specifications used to create the corpus, while
`pypto/` and `pypto-lib/` contain the actual solver inputs.
