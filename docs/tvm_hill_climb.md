# Apache TVM hill-climb baseline

## Scope and provenance

`TvmHillClimbSolver` is an independently written behavioral reimplementation of
Apache TVM USMP's graph-guided ordering search. The pinned reference is
[`hill_climb.cc` at v0.14.0](https://github.com/apache/tvm/blob/v0.14.0/src/tir/usmp/algo/hill_climb.cc),
introduced by [apache/tvm#9704](https://github.com/apache/tvm/pull/9704).
TVM's planner boundary is described by the
[`Unified Static Memory Planning RFC`](https://github.com/apache/tvm-rfcs/blob/8e5c1250a6632033c8ffa2d901b3a4b0ce59f982/rfcs/0009_Unified_Static_Memory_Planning.md).

Apache TVM is Apache-2.0 licensed. This repository neither vendors nor links
TVM; `NOTICE` and `THIRD_PARTY_NOTICES.md` retain attribution.

## Frozen policy

The baseline:

1. orders buffers by decreasing size, conflict degree, reverse name, and stable
   representative;
2. decodes the order with greedy lowest-gap placement;
3. chooses a buffer ending at the current high-water mark;
4. collects its earlier conflict neighbors and their earlier conflict
   neighbors;
5. swaps one first-level node with a distinct first- or usually second-level
   node;
6. accepts improvements and occasionally a worse peak with probability decaying
   by attempt number; and
7. stops at the attempt budget or optional target peak.

This is graph-guided ordering search with a small annealing component, not
direct hill climbing over offsets. Every candidate offset set is regenerated by
the shared decoder.

## Deliberate differences

- `std::mt19937_64` and an explicit seed replace platform-specific random state.
- The best placement is retained after accepting a worse transition.
- Pinned nodes are fixed and colocations become one search node.
- The shared decoder handles alignment and independently validates results.
- Complete over-capacity candidates may guide search toward feasibility.
- The target is optional; without it the full attempt budget is consumed.

These changes improve determinism and anytime behavior without changing the
named neighborhood. Results should still be described as a behavioral, not
bit-for-bit, reimplementation.

## Structured inputs and limitations

For structured inputs, graph edges include temporal conflicts, explicit
separations, and exclusive pins; the decoder handles fixed pools, reservations,
multi-interval liveness, temporal exclusions, and reuse-cost objectives.
Flexible pool assignment and bank-cost objectives remain unsupported.

Ordering search cannot reach every useful offset arrangement. The TVM move can
also stall when a high-water node has too few earlier movable conflict
neighbors. Keep this solver frozen as an ablation; direct offset/region moves
belong in a separately named research solver.
